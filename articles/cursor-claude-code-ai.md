# Cursor 進入「戰時狀態」：當 Claude Code 用 25 倍補貼搶市場，AI 寫程式工具的未來在哪裡？

> 本文是 YouTube 影片「Cursor 進入戰時狀態！Claude Code 用 25 倍補貼搶市場」的延伸閱讀版。
> 影片講了核心觀念，本文加上獨家分析和趨勢預判。
> 影片版本：https://www.youtube.com/watch?v=MeVRf96xkt8

## 影片重點回顧

Cursor 在今年一月開了一場全體員工大會，簡報標題叫「戰時狀態」。起因是員工試用 Anthropic 最新的 Claude Opus 後發現，AI 程式碼生成已經強到開發者可以直接下指令、收成品，不需要逐行審查。Cursor「程式設計師專用的 Google 文件」核心定位直接被動搖。

數字最殘酷：Claude Code 上線六個月年化營收就達 25 億美元，正式超越 Cursor 的 20 億。更驚人的是，Claude Code 每月收 200 美元，實際運算成本高達 5,000 美元，等於 Anthropic 正在用 25 倍的幅度補貼用戶。

## 【獨家】AI 寫程式工具的四個世代演進

![Close-up of a futuristic robot toy on a wooden floor, showcasing modern innovation.](images/ai-programming-tools.jpg)
*Photo by [Kindel Media](https://www.pexels.com/@kindelmedia) on [Pexels](https://www.pexels.com/photo/a-white-and-black-robot-toy-8982665/)*


要理解 Cursor 為什麼會恐慌，你需要看到整個 AI 寫程式工具的演進軌跡：

**第一代：自動補全（2021-2023）**
GitHub Copilot 開啟了這個時代。工程師寫一行，AI 猜下一行。本質上是「更聰明的 Tab 鍵」，工程師仍然完全掌控程式碼。

**第二代：協作式編輯（2023-2025）**
Cursor 就在這一代崛起。工程師跟 AI 並肩工作，可以用自然語言描述需求，AI 生成程式碼片段，工程師審查後合併。Cursor 的產品邏輯就建立在這一代的假設上：人機協作。

**第三代：代理式開發（2025-現在）**
Claude Code、Codex 代表的就是這一代。工程師不再逐行看程式碼，而是向 AI 代理下達任務，代理自主完成。工程師變成「品質管理者」而不是「程式碼撰寫者」。

**第四代：全自主開發（未來）**
AI 從需求文件直接產出可部署的軟體。工程師的角色轉向產品設計、架構決策和商業判斷。

Cursor 恐慌的原因就在這裡：他們的產品完美契合第二代，但第三代已經來了，第四代正在路上。如果不轉型，他們的產品就會像翻譯社遇到完美的機器翻譯一樣，存在的理由消失。

## 【獨家】補貼戰的歷史教訓：誰活到最後？

![A strategic arrangement of colorful pawns connected on a game board, symbolizing networking and teamwork.](images/strategy-anthropic.jpg)
*Photo by [Pixabay](https://www.pexels.com/@pixabay) on [Pexels](https://www.pexels.com/photo/close-up-photography-of-yellow-green-red-and-brown-plastic-cones-on-white-lined-surface-163064/)*


Anthropic 用 25 倍補貼搶市場，這個策略其實不新鮮。科技產業的歷史上，補貼戰已經打過無數次：

**滴滴 vs Uber 中國（2014-2016）**
雙方每月燒掉數億美元補貼乘客和司機，最終 Uber 退出中國市場，滴滴合併後開始調價。

**Netflix vs 串流大戰（2019-2024）**
Netflix、Disney+、HBO Max 同時燒錢做原創內容，用低價訂閱搶用戶。結果是全部漲價，部分平台縮減內容。

**現在的 AI 寫程式工具戰**
Anthropic 用 Google 和 Amazon 的資金補貼 Claude Code，OpenAI 有微軟的支持推 Codex，Cursor 沒有同等規模的後台。

歷史的規律是：補貼戰最後活下來的，要嘛是「資金最雄厚的那個」，要嘛是「找到差異化定位的那個」。

Cursor 選擇的路是兩條並行：自建模型（擺脫依賴）加上轉向企業市場（穩定收入）。企業合約從 13.6% 成長到六成，客戶包括 Meta 和 Nvidia。這條路能不能走通，取決於他們的自有模型能不能追上 Claude 和 GPT 的水準。

## 【獨家】台灣企業的啟示：你的「Cursor 時刻」來了嗎？

![A modern humanoid robot with digital face and luminescent screen, symbolizing innovation in technology.](images/business-cursor.jpg)
*Photo by [Kindel Media](https://www.pexels.com/@kindelmedia) on [Pexels](https://www.pexels.com/photo/low-angle-shot-of-robot-8566526/)*


我服務過超過 400 家企業做 AI 培訓，其中不少科技業。跟他們交流後，我發現 Cursor 的故事對台灣企業有三個直接的啟示：

**啟示一：不要把護城河建在別人的平台上**

如果你的核心產品依賴第三方的 AI 模型，你要想清楚：當模型廠商自己來做你的業務時，你有什麼籌碼？Cursor 依賴 Anthropic 和 OpenAI 的模型，結果 Anthropic 推出 Claude Code 直接搶它的客戶。

台灣的 SaaS 公司、AI 應用開發商都應該問自己這個問題。

**啟示二：AI 工具的選擇不是一次性決策**

Valon 九十多名員工一夜之間從 Cursor 轉向 Claude Code。在 AI 時代，工具切換的成本越來越低，忠誠度越來越薄。企業不應該把所有雞蛋放在同一個 AI 工具籃子裡。

**啟示三：真正的護城河是「人」不是「工具」**

工具會一直變，Claude Code 今天很強，明天可能被另一個東西取代。但你的團隊理解 AI 的能力、知道怎麼用 AI 解決業務問題的這個能力，是別人搶不走的。

這也是為什麼我一直說：你是機長，AI 是機組人員。

## 阿峰觀點

這場 AI 寫程式工具大戰，表面上是 Cursor vs Claude Code 的競爭，底層其實是一個更根本的問題：**當 AI 強到不需要中間人的時候，所有「中間層」的產品都會面臨存亡危機。**

Cursor 至少有自覺，他們的核心價值觀叫「刪除這款產品」。大多數公司連意識到危機都做不到。

對我們每個人來說，最重要的不是糾結該用 Cursor 還是 Claude Code，而是持續提升自己駕馭 AI 工具的能力。因為不管工具怎麼變，會用工具的人永遠比不會用的人有優勢。
