## 重點回顧

你有沒有這種經驗？花了一個小時寫完一篇文章，然後還要再花一個小時把它改成 Facebook 版本、Instagram 版本、Threads 版本、LINE 社群版本，最後再發到部落格。每個平台的格式不一樣、語氣不一樣、字數限制不一樣，光是複製貼上再調整就讓人崩潰。

我以前也是這樣。每次發一篇內容，從構思到全平台上線，至少要 2 個小時。一週發三次，光是「發文」這件事就吃掉 6 小時。這還不包括製作輪播圖、寫 SEO 描述、設定排程這些瑣碎的事。

直到我用 Claude Code 建了一套叫做「Post Factory」的自動化流程。

現在，我只要下一個指令，15 分鐘內就能完成以下所有事情：一篇 2000 字的深度文章、五個平台各自優化的貼文、六張品牌風格一致的輪播圖、一篇 SEO 優化的部落格文章，以及自動排程發布。

這不是未來式，這是我每天在用的工作流程。這篇文章會完整拆解這套系統的每一個步驟，讓你也能建立自己的內容自動化流程。


## 【獨家】Post Factory 完整流程拆解

Post Factory 的核心概念很簡單：把「寫一篇內容」到「全平台發布」的整個過程，拆成 7 個可自動化的步驟，然後用 Claude Code 的 Skill 系統串起來，一個指令觸發全部。

### 步驟一：搜尋資料

每篇文章的起點不是「我想寫什麼」，而是「現在什麼話題有價值」。

Post Factory 會先用 Gemini 的即時搜尋功能，去抓取你指定主題的最新趨勢、數據和新聞。比如我今天想寫「Claude Code 教學」，它會自動搜尋 Claude Code 最新的功能更新、社群討論熱度、相關的使用案例。

這一步的關鍵是「用 AI 搜尋」而不是自己 Google。AI 搜尋會直接整理出結構化的資訊，省掉你開 20 個分頁、一個一個讀的時間。

### 步驟二：寫文章

拿到搜尋結果後，Claude Code 會根據事先設定好的「Content Brief」來寫文章。Content Brief 包含目標關鍵字、文章架構、目標讀者、語氣風格這些參數。

這裡有一個很重要的設定：語氣風格。我的 Content Brief 裡面寫死了「像教練說話，不像教授」。所以 AI 產出的文章不會是那種學術論文的口吻，而是像我平常講話的風格。

一篇 2000 字的文章，大概 2-3 分鐘就寫完了。但寫完不代表能直接用，後面還有品質把關的步驟。

### 步驟三：萃取素材

文章寫完後，Post Factory 會自動從裡面萃取「可重複使用的素材」：金句、關鍵數據、核心論點、適合做圖的段落。

這些素材會被整理成一個結構化的 JSON，後面的步驟會用到。比如金句會變成輪播圖上的標題，關鍵數據會變成 Facebook 貼文的 hook，核心論點會變成 Threads 的對話起點。

### 步驟四：四平台文案

這是最花時間的步驟之一，也是自動化帶來最大價值的地方。

同一個訊息，在不同平台要用不同的方式講。Post Factory 會根據每個平台的特性，自動生成差異化的文案：

Facebook 版本是長文帶故事，適合深度閱讀。開頭用一個吸引人的問題或場景，中間展開論述，結尾帶 CTA。

Instagram 版本精煉到 2200 字以內，重點放在前三行（因為 IG 會折疊），搭配 30 個精選 hashtag。

Threads 版本走口語對話風格，像是跟朋友聊天。短句、有觀點、容易引發回覆。

LINE 社群版本最簡潔，重點加連結，因為 LINE 群組的閱讀習慣是快速掃過。

四個版本同時產出，不到一分鐘。

### 步驟五：六張輪播圖

這是 Post Factory 最讓人驚豔的部分。它會用 Python 的 PIL 函式庫，根據品牌設計系統自動生成六張 1080x1080 的輪播圖。

每張圖都嚴格遵守 CVI 品牌規範：色碼、字體、版面結構、浮水印位置。不會出現「這張圖看起來不像我們的風格」的問題。

六張圖的結構也是固定的：封面頁（大標題 + 副標題）、數據頁（關鍵數字對比）、步驟頁（流程拆解）、深度頁（核心概念展開）、行動頁（讀者可以做什麼）、結尾頁（總結 + CTA）。

### 步驟六：部落格文章

文章會被自動轉換成 Hugo 部落格的格式，包含 YAML frontmatter（標題、描述、關鍵字、分類、標籤）和正文。

SEO 優化也是自動的：meta description 控制在 155 字以內、標題包含目標關鍵字、內文自然分布關鍵字。文章寫完後直接 push 到 GitHub，Hugo 自動建置，部落格就更新了。

### 步驟七：自動發文

最後一步是透過 n8n 自動化引擎，把所有內容排程發布。Facebook 和 LINE 透過 API 直接發，Instagram 透過排程工具預約，Threads 手動發（因為 API 限制）。

整個流程從「下指令」到「全平台上線」，15 分鐘內完成。如果內容不需要太多修改，甚至可以壓縮到 10 分鐘以內。


## 【獨家】三個讓自動化不犧牲品質的秘訣

很多人聽到「AI 自動化」就會擔心品質問題。說實話，如果只是把 ChatGPT 的輸出直接貼上去，品質確實會有問題。Post Factory 的設計從一開始就把品質控管放在核心位置。

### 秘訣一：品牌一致性靠系統，不靠記憶

Post Factory 會讀取一個叫做「Design System」的設定檔，裡面定義了所有品牌規範：主色碼 #00D4FF、背景色 #0A1628、字體 Noto Sans TC、語氣風格「像教練說話」、禁用詞彙清單。

每次產出內容時，AI 不是「憑感覺」模仿你的風格，而是「照規範」執行。這確保了不管是第一篇還是第一百篇，品牌調性都是一致的。

### 秘訣二：跨平台差異化靠模板，不靠重寫

每個平台都有一個預設的「文案模板」，定義了開頭結構、字數範圍、語氣參數、hashtag 策略。AI 在生成文案時，是「套模板 + 填內容」，而不是「從零開始寫一篇新的」。

這樣做的好處是：每個平台的文案都不一樣（避免被演算法懲罰），但核心訊息是一致的（確保品牌傳播的統一性）。

### 秘訣三：原創性靠觀點，不靠文字

AI 最容易被詬病的就是「寫出來的東西都差不多」。Post Factory 的解決方案是：每篇文章都必須包含一個「阿峰觀點」段落，這個段落的內容來自我自己的經驗和看法，不是 AI 生成的。

AI 負責框架、資料整理、格式轉換這些機械性的工作。觀點、立場、個人故事這些「只有你才能寫」的東西，還是要自己來。

這就是我常說的：你是機長，AI 是機組人員。機長決定飛往哪裡，機組人員負責確保飛行過程順利。


## 阿峰觀點

我用 Post Factory 大概三個月了，最大的感受不是「省時間」，而是「降低了發文的心理門檻」。

以前要發一篇文，心裡會想：又要寫五個版本，又要做圖，又要排程。光想就累了，乾脆明天再說。結果就是一拖再拖，一週發不了幾篇。

現在不一樣了。因為知道只要 15 分鐘就能搞定，所以更願意發。發文頻率從一週兩三次變成每天一篇，內容的多樣性也提高了，因為不用擔心「寫這個要花太多時間」。

但我要特別強調一件事：自動化不是萬能的。

Post Factory 能幫你處理 80% 的機械性工作，但剩下的 20% — 你的觀點、你的經驗、你對讀者的理解 — 這些是 AI 替代不了的。如果你只是把 AI 的輸出原封不動地貼出去，讀者很快就會發現。

我的建議是：先從小地方開始。不用一步到位建一整套 Post Factory，先試試用 Claude 整理一份會議記錄，或者幫你潤稿一封 Email。等你熟悉了 AI 的能力和限制，再慢慢擴展到更複雜的工作流程。

會用、懂用、好用、每天用。這四個階段，急不來。
