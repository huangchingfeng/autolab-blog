10 個素人用 Claude Code 做出的真實產品｜最快 4.5 小時上線

Claude CodeVibe CodingAI開發實戰案例

10 個素人用 Claude Code 做出來的真實產品，最快只花 4.5 小時

一句話總結：不是工程師也能用 AI 寫出上架的 APP、自動化系統、甚至賣到國外的產品——這 10 個人的故事，會讓你重新定義「會寫程式」這件事。

前言

2026 年 3 月 5 日晚上，我們辦了第一場 Claude Code 使用者群組的線上分享會。

本來只是想讓大家聊聊「你最近用 Claude Code 做了什麼」，結果一開麥就停不下來。遊戲製作人、設計師、廣告導演、露營旅館老闆、永續顧問⋯⋯背景天差地遠，但每個人都用 Claude Code 做出了「以前做不到」的東西。

這不是一篇教你怎麼安裝 Claude Code 的教學文。這是 10 個真實的人，用 AI 解決真實問題的故事。每一個案例，都是「你是機長，AI 是機組人員」這句話的最佳示範。

關鍵重點

  • 非技術背景的人，產出不輸工程師：設計師 7 天漲 1000 粉、廣告導演自己刻出 ERP
  • Spec 給得好，4.5 小時就能做出 6000 人同時在線的系統
  • 從最小痛點開始，比一開始就想做大系統有效 100 倍

案例 1：遊戲製作人的虛擬 KOL 工廠

Photo by RDNE Stock project on Pexels

第一位分享者有 15 年遊戲製作經驗，轉行到諮詢業當特助。公司需要批量製作行銷素材，還要培養 5 位虛擬 KOL。

他自己本地架了圖片生成設備，用 Claude Code 從零刻出一整套工作流管理系統：輸入目標 → AI 拆解任務 → 自動生產 → 追蹤狀態 → 發布 → 生成文案。

另外還串接了 IG API 做數據分析，找出爆款因素，未來要連動到素材生產建議。

「如果今天你想做的東西有點大，先從最麻煩的那個痛點開始，用最小方式去解決。」

他最寶貴的經驗是：用 CLAUDE.md 記錄每次踩坑和改版原因。因為之前 AI 會把 A 改成 B，又把 B 改回 A。有了文字記錄，這種循環就不會再發生。

案例 2：永續產業的低成本 AI 系統

Photo by Airam Dato-on on Pexels

阿嶽做永續顧問，他面對的現實是：永續部門通常沒有預算買軟體。

所以他寫了一套可以放進任何 AI 就能用的多代理人系統，搭配課程教客戶使用。重點是「人機協作」——自動化太強，客戶反而跟不上。

這套系統已經在全台最大的永續能源研究基金會底下的協會開課，而且已經在變現了。

實戰心法

  1. 不要追求全自動：套裝軟體很強沒有錯，但人的使用經驗沒跟上來會很麻煩
  2. 搭配課程一起賣：工具 + 教育 = 更好的商業模式
  3. 從真實集合經驗出發：人機協作的 know-how 來自實際使用

案例 3：Eason 的五合一 AI 產品線（最狠的一位）

Photo by Google DeepMind on Pexels

Eason 前京城資訊副總監、數據長，現在一個人用 Claude Code 做出了五個產品：

心靈好友 APP：針對心理健康設計，用雙語言模型互相檢測，避免被單一 AI 繞過安全機制。已上架 App Store，一個人 80 小時內完成。

尾牙抽獎系統：支援 6000 人同時在線。從寫第一個 Spec 到功能可用，總共 4.5 小時。已開源放 GitHub。

HR 招募 SaaS：自動萃取職缺技能 → 媒合履歷 → 發邀請 → 自動出考卷 → 回傳分數。正在找客戶測試。

企業智能顧問：結合 ERP/CRM，內建 30+ 分析工具，落地部署，資料完全隔離。

舊系統翻新：讓 AI 逆向工程讀舊程式碼，產出完整 Spec，再用新語言重寫。幾乎 one-shot 就能達到功能一致。

「Spec 不是只有產品的 Spec，還包含你的部署 Spec、資安要求、消耗的要求。先讓 AI 建好 ToolChain，再開始開發。」

案例 4：設計師 Abi 的社群自動發文引擎

Abi 完全是設計和行銷背景，不是技術出身。她用 Claude 的 Skill 功能建了一套社群自動發文系統，串接 Threads。

關鍵在 Skill 的品質——她已經寫了 20+ 個 Skill，組合成一個「小編人格」。結果新帳號 7 天從 600 漲到 1600 粉，現在已經 2400。

「大家都在說 AI 寫文會降低消費者信任度。我認為不是 AI 的問題，是你的內容有沒有打中讀者。」

她接下來要做的是：打造一個有 IP 形象的虛擬小編，讓 AI 自動產出精緻的圖文內容。

案例 5-10：更多瘋狂的實戰案例

Mark（Adobe 代理商）：20 個 Engine、200+ Skill，打鼓遊戲網站已經賣到國外變現。還做了 Chrome 插件讓你開車時用藍牙控制 YouTube 播放速度。自稱「嚴重過度工程化」，但花大量時間打磨環境後，「好像什麼都可以做」。

文凱：AI 文案生成系統，客戶輸入 1-2 句話就能產出完整的 SEO 長文和社群貼文。研究了大量「什麼是好內容」的隱性知識。

Frank（自由接案）：把同一個問題丟給 4 家 AI（Claude、GPT、Grok、Gemini），用 Claude 整合成報告，投國外創新大賽獲得回應。深入研究 AI 幻覺的底層原因和歸因系統。

TK（日本露營旅館老闆）：逆向破解日本訂房系統 API、做了 AEO 網站優化工具讓中小企業被 AI 爬蟲找到（第一天就 13 家店家使用）、還建了 5000+ Bug 案例庫。在鄉下寫 code，計畫把台灣的 AI 教育帶到日本。

Moco：做了 Chrome 插件，把英文 YouTube 影片自動翻譯成中文配音，開車時可以聽。用 Whisper + OpenAI TTS 實現。

Miru：用 Notion 串 Gemini 做半自動內容工作流，解決客戶素材給太慢的問題。

技術討論亮點：Lovable vs Claude Code

分享會上有一段很實用的技術討論，比較 Lovable 和 Claude Code：

  • Lovable：對新手最友好、UI 最好看、一鍵部署。適合做前台原型
  • Claude Code：完全自由、可以做複雜後台。需要自己處理部署

TK 的組合拳：用 Lovable 建漂亮的前台 → 同步到 GitHub → Claude Code 開發後台 → n8n 串接自動化

這個組合讓每個工具各司其職，是目前社群公認最有效率的做法。

阿峰觀點

聽完這 10 個人的分享，我最大的感觸是：AI 真的把「能不能做」變成了「想不想做」。

廣告導演不懂程式？沒關係，Claude Code 幫你刻。永續部門沒預算？沒關係，寫一套放進免費 AI 就能用的系統。在日本鄉下？沒關係，照樣寫出讓 13 家店搶著用的工具。

但我也看到一個共同點：做得好的人，都不是「叫 AI 幫我做」，而是「跟 AI 一起做」。

Spec 給得完整、Skill 設計得精準、踩坑記錄寫得清楚——這些「人」做的準備工作，才是決定 AI 產出品質的關鍵。

這就是我一直在講的：「你是機長，AI 是機組人員。」機組人員再厲害，機長不會飛，飛機也到不了目的地。

建議行動

  1. 今天就開始你的第一個最小痛點：找到你工作中最煩的一件事，用 Claude Code 解決它（30 分鐘）
  2. 建立你的 CLAUDE.md：把踩坑記錄、改版原因都寫下來，避免 AI 反覆犯錯（15 分鐘）
  3. 試試 Lovable + Claude Code 的組合：前台用 Lovable 快速出原型，後台用 Claude Code 開發（1 小時）

結語

這只是第一場分享會，接下來我們會定期舉辦。如果你也想加入這個社群，或者你已經用 Claude Code 做出了什麼東西，歡迎來分享。

記住：不要等到「學會了」才開始做。先做，遇到問題再學。 這 10 個人都是這樣走過來的。
